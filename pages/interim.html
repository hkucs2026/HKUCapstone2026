Layer Architecture

Four-layer structure used to separate interface, orchestration, intelligence, and persistence
Architecture decision: the layered design keeps UI concerns, backend orchestration, AI processing, and storage separate so each part can evolve without tightly coupling the whole system.

Presentation

Backend Orchestration
FastAPI coordinates authentication, request handling, reminder logic, health probes, and service integration.
AI Processing
The AI layer combines Moonshot Kimi K2.6, FAISS-backed retrieval, safety classification, and structured answer generation.
Persistence
PostgreSQL stores user and appliance records, while FAISS supports vector search over the curated corpus.

Pipeline Structure

Main request flow implemented for multimodal diagnosis and grounded response generation
Diagnostic Request Flow
  • User submits text and optionally an image from the iOS Fix flow.
  • FastAPI receives the request, validates it, and passes it into the orchestration layer.
  • The VLM layer extracts appliance type, brand, model number, error codes, and damage indicators from the image.
  • The retrieval stage queries the manual corpus and related knowledge base content for grounded context.
  • The answering LLM produces a structured result containing fault cause, risk rating, and action plan.
  • The deterministic safety guardrail decides the response severity before the final result is returned to the frontend.
AI Routing Structure
Mode Name Description
A Pure RAG Corpus retrieval only, no web fallback
B Hybrid + Web Fallback Dense + BM25 with deterministic threshold-based web search
C Agentic (Production) LLM decides per query whether to use corpus, web, or parametric knowledge
D Parametric-Only LLM training knowledge only, no retrieval
W Web-Only Live web search only, no corpus retrieval

AI Diagnostic Response on Fix Screen
Response Element What It Shows
Problem Overview Short summary of the visible issue detected from the uploaded photo and user description
Cause of Damage Likely explanation of why the issue happened, with grounded observations from the image
Repair Time Estimated time needed for the repair task so users can judge difficulty and effort
Risk Level Severity output such as low, moderate, or high, used by the safety guardrail before guidance is shown
Issue Category Classification label such as walls and surfaces to keep the response structured and easier to route
Follow-Up Action Safe next-step guidance, which can remain DIY-oriented or escalate to professional referral when risk is high
Safety Guardrail Design
Risk Level Examples System Response
Low Clogged filters, cosmetic replacements, basic resets Step-by-step DIY guidance with standard safety notes
Moderate Minor plumbing resets, external component assembly Conditional DIY guidance with enhanced safety reminders and PPE recommendations
High Internal electrical wiring, gas-line issues, sealed refrigerant High-risk banner, immediate professional referral protocol, and no DIY guidance
